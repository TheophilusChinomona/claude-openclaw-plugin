# Workspace Wizard Phase Checklist

## Phase 0: Setup
- [ ] Workspace name determined
- [ ] Workspace mode selected (single/multi-agent)
- [ ] Workspace path confirmed
- [ ] Resume detection completed

## Phase 1: Discovery
- [ ] Primary purpose defined (one sentence)
- [ ] Target audience described
- [ ] Communication channels identified
- [ ] Single vs multi-agent decided
- [ ] Use cases documented (3-5 scenarios)
- [ ] Refusal boundaries identified
- [ ] Required files determined
- [ ] Memory strategy recommended

## Phase 2: Memory Design
- [ ] Memory strategy confirmed (minimal/standard/comprehensive)
- [ ] MEMORY.md categories defined
- [ ] Daily log format agreed (if applicable)
- [ ] Team memory structure planned (if multi-agent)

## Phase 3: Soul Design
- [ ] Identity statement crafted
- [ ] 3-5 core values with behavioral predictions
- [ ] Boundaries specified (never/confirm/fallback)
- [ ] Voice designed (tone, length, format, anti-patterns)
- [ ] Token budget check (under 2,000 words)

## Phase 4: Workspace Design
- [ ] IDENTITY.md created
- [ ] AGENTS.md designed (memory, safety, group, workflows, subagent rules)
- [ ] Optional files scoped (USER, GOALS, HEARTBEAT, BOOTSTRAP, TOOLS)
- [ ] Multi-agent routing configured (if applicable)

## Phase 5: Scaffolding
- [ ] Directory structure created
- [ ] All files generated from design decisions
- [ ] Validation checks passed (word counts, line counts, cross-refs)
- [ ] No secrets in workspace files

## Phase 6: Quality Audit
- [ ] Scored audit completed
- [ ] P0 issues resolved
- [ ] P1 issues documented for follow-up
