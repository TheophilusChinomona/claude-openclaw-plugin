# Reference Agent Templates

Starting points for building OpenClaw agent workspaces. These templates come from [agency-agents](https://github.com/msitarzewski/agency-agents) and provide well-structured agent personas you can adapt.

## Available Templates

| Template | Role | Best For |
|----------|------|----------|
| [software-architect](software-architect.md) | System design, DDD, architectural decisions | Technical leadership agents |
| [code-reviewer](code-reviewer.md) | Constructive code review, quality assurance | Development workflow agents |
| [security-engineer](security-engineer.md) | Threat modeling, vulnerability assessment | Security-focused agents |
| [technical-writer](technical-writer.md) | Developer docs, API references, tutorials | Documentation agents |
| [devops-automator](devops-automator.md) | Infrastructure, CI/CD, cloud operations | Operations agents |
| [sprint-prioritizer](sprint-prioritizer.md) | Agile planning, feature prioritization | Product management agents |
| [ux-researcher](ux-researcher.md) | User research, usability testing | Research and insight agents |
| [feedback-synthesizer](feedback-synthesizer.md) | Multi-channel feedback analysis | Customer intelligence agents |

## How to Use

During the OCW **Agent Discovery** or **Design Soul** workflows, reference these templates to:
1. Borrow personality traits and communication styles
2. Adapt core missions and principles for your agent's domain
3. Use the critical rules patterns as starting points for SOUL.md boundaries
4. Translate the vibe/identity into OpenClaw IDENTITY.md format
